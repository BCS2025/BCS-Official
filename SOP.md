# 比創空間客製化訂單系統 - 標準作業流程 (SOP)

份文件旨在建立「網站管理者 (您)」與「系統開發者 (我)」之間的協作標準，確保在系統架構不變動的前提下，能順暢執行商品上架與庫存管理。

---

## 1. 角色職責分工

| 角色 | 負責範圍 | 主要工具 |
| :--- | :--- | :--- |
| **管理者 (您)** | **數據與營運**<br>1. 庫存數量管理<br>2. 訂單接收與出貨<br>3. 提供新商品規格與圖片 | **Google Sheet (後台)**<br>LINE / Email |
| **開發者 (AI)** | **程式與結構**<br>1. 網站程式碼維護 (Frontend)<br>2. 定義新商品邏輯與 SKU<br>3. 系統功能修復 | **VS Code (程式碼)**<br>GitHub |

---

## 2. 庫存維護流程 (日常營運)

此流程**不需要**通知開發者，您可隨時獨立完成。

### 補貨 / 調整庫存
1.  開啟 **Google Sheet 後台資料庫**。
2.  切換至底部的 **`Inventory` (庫存)** 分頁。
3.  找到對應商品的 **`SKU`** (A欄)。
    *   *小撇步：可參考 C 欄的 `ProductName` 中文備註。*
4.  直接修改 **`Stock`** (B欄) 的數字。
    *   輸入 `50` = 設定庫存為 50。
    *   輸入 `0` = 設為缺貨 (網站會顯示「缺貨中」或無法加入購物車)。

### 注意事項
*   若某商品 **未列在 Inventory 表上**：系統預設為 **「無限庫存」** (接單無上限)。
*   請勿隨意修改 **A欄 (SKU)** 的內容，否則網站會找不到對應資料。

## 3. 新商品上架流程 (協作)

此流程需要雙方配合。

### Step 1: 提出需求 (您 -> 我)
請提供以下資訊給我：
1.  **商品名稱** (例如：皮革手環)
2.  **商品定價** (以及是否有「客製化加價」邏輯)
3.  **商品圖片** (請上傳圖片給我，或提供檔名)
4.  **規格選項** (例如：顏色[紅/黑]、尺寸[S/M/L]、是否刻字)
5.  **商品描述** (文案)

### Step 2: 系統實作 (我)
1.  我會將商品資訊寫入網站程式碼 (`src/data/products.js`)。
2.  我會設定好該商品的 **SKU 命名規則**。
3.  **交付項目**：通知您更新完成，並提供一份 **「SKU 對照清單」**。

### Step 3: 更新庫存 (您)
1.  收到「SKU 對照清單」後，請打開 Google Sheet `Inventory` 分頁。
2.  將新的 SKU **複製貼上** 到表格的最下方 (A欄)。
3.  在 B 欄填入目前的初始庫存量。
4.  在 C 欄填入中文名稱 (方便您自己辨識)。

## 4. 訂單處理流程

當客人下單後：
1.  **通知**：
    *   您的 **LINE** 會收到即時推播 (含訂單摘要)。
    *   客人的 **Email** 會收到確認信 (含匯款資訊)。
2.  **查單**：
    *   打開 Google Sheet **`Orders` (訂單)** 分頁。
    *   最新訂單會顯示在最下方。
    *   確認欄位：`總金額`、`運送方式`、`對稿需求`。
3.  **確認收款**：
    *   確認收到匯款後，即可開始製作。
4.  **出貨**：
    *   依據 `運送詳情` 地址出貨。
    *   (目前系統尚未串接自動出貨通知，需人工用 LINE 告知客人已出貨)。

## 5. 常見問題排除 (Q&A)

**Q: 為什麼網站上顯示「缺貨」，但我倉庫明明還有貨？**
A: 請檢查 Google Sheet `Inventory` 分頁，確認該 SKU 的 `Stock` 數字是否大於 0。

**Q: 我在 Sheet 上面自己加了一行 SKU，為什麼網站上沒出現那個商品？**
A: 因為「網站顯示」需要改程式碼 (Frontend)。單純在 Sheet 加 SKU 只是「準備好倉庫空位」，您必須先執行 **「新商品上架流程」** 通知我寫 Code。

**Q: 客人說他選了「紅色」，但 Email 訂單明細顯示英文代碼？**
A: 請通知我修正。這通常是顯示標籤 (Label) 的轉換邏輯需要更新。
