本文件為「網站管理者 (您)」與「系統開發者 (AI)」之間的協作標準。

## 3. 訂單處理流程

### 當客人下單後：

1. **通知**
   - 您的 **LINE** 會收到即時推播（含訂單摘要）
   - 客人的 **Email** 會收到確認信（含匯款資訊）

2. **查單**
   - 開啟後台 → **訂單管理** (`/admin/orders`)
   - 最新訂單排在最上方
   - 點擊「詳情」可查看：客戶資訊、訂購商品規格、運送方式、應付金額

3. **確認收款**
   - 客人匯款後會在 LINE 告知訂單編號與匯款後五碼
   - 確認無誤後，將訂單狀態改為 **「已付款」**

4. **製作 & 出貨**
   - 製作完成後，將狀態改為 **「已出貨」**
   - 目前需人工用 LINE 告知客人已出貨（尚未串接自動通知）

### 訂單狀態說明

| 狀態 | 意義 |
| :--- | :--- |
| 待付款 | 訂單剛成立，等待客人匯款 |
| 已付款 | 確認收到款項，可開始製作 |
| 已出貨 | 商品已寄出 |
| 已完成 | 客人確認收到 |
| 已取消 | 訂單取消 |

### 批次操作
- 可勾選多筆訂單 → 批次移至垃圾桶（封存）
- 垃圾桶內訂單可還原或永久刪除

## 4. 商品管理

### 新商品上架（管理者可獨立操作）

1. 開啟後台 → **商品管理** (`/admin/products`)
2. 點擊「新增商品」
3. 填寫以下欄位：

| 欄位 | 說明 |
| :--- | :--- |
| 商品 ID | 格式：`prod_xxx`（英文小寫+底線，建立後不可更改） |
| 商品名稱 | 顯示在網站上的名稱 |
| 定價 | 基本售價 |
| 優惠價 | 勾選「特價中」後啟用（原價劃線顯示） |
| Slogan | 商品標語（顯示在圖片下方） |
| 短版敘述 | 20字以內，顯示在首頁商品卡片 |
| 排序權重 | 數字越大越前面（精選商品設高數字，如 100） |
| 商品啟用 | 關閉後商品不在前台顯示 |

### 商品圖文說明

| 欄位 | 字數 | 顯示位置 |
| :--- | :--- | :--- |
| 短版敘述 (`description`) | 建議 20 字內 | 首頁卡片 + 商品頁標題下 |
| 詳細介紹 (`detailed_description`) | 無限制 | 商品頁左下方說明區塊 |

### 商品排序邏輯

- **排序依據**：`sort_order` 欄位（數字越大越靠前）
- **精選商品**：設定高數字（例：100、90）
- **一般商品**：使用預設值（10）
- 相同 `sort_order` 者，依**上架日期**由新到舊排列

### 需要開發者協助的情況

以下情形需通知開發者：
- 複雜的多層條件定價邏輯（例：「雙色+大尺寸才加價 XX 元」）
- 商品選項需要「連動顯示/隱藏」邏輯（conditional fields）
- 訂製獨特的計價公式

## 5. 原料庫存管理

> **注意**：此處管理的是「製作商品所需的原料」（例：木板、壓克力板），不是商品本身的庫存。

### 日常維護（不需通知開發者）

1. 開啟後台 → **原料庫存** (`/admin/inventory`)
2. 找到對應原料，點擊「編輯」
3. 修改「目前庫存」數字後按儲存

### 欄位說明

| 欄位 | 說明 |
| :--- | :--- |
| 原料名稱 (ID) | 原料識別代號（英文小寫+底線） |
| 目前庫存 | 目前實際持有數量 |
| 安全庫存 | 低於此值會觸發告急標示 |
| 狀態 | 充足 / 庫存告急（紅色警示） |

### 低庫存告警

- 當目前庫存 ≤ 安全庫存時，狀態欄顯示**紅色「庫存告急」**
- 可點擊「測試告急通知」按鈕，手動觸發 LINE 推播通知

### 新增原料

1. 點擊「新增原料」
2. 填入：原料名稱（中文）、ID 代號（英文小寫+底線）、目前庫存、安全庫存
3. 儲存

## 6. 優惠券管理

> **新功能（SOP 舊版未包含）**：完整的折扣碼系統。

### 建立優惠券

1. 開啟後台 → **優惠券管理** (`/admin/coupons`)
2. 點擊「新增優惠券」
3. 設定參數：

| 欄位 | 說明 |
| :--- | :--- |
| 優惠碼 | 客人輸入的折扣代碼（可自訂或隨機產生） |
| 折扣類型 | 百分比折扣 / 固定金額折扣 / 免運費 |
| 折扣值 | 百分比（如 10 = 9折）或金額（如 50 = 折 50 元）|
| 最低消費 | 訂單需達此金額才可使用（0 = 無限制）|
| 適用範圍 | 全商品 / 指定商品 |
| 使用次數上限 | 空白 = 無限制 |
| 有效期限 | 設定起訖日期 |
| 啟用狀態 | 關閉後折扣碼失效 |

### 注意事項
- 折扣碼大小寫**不敏感**（系統自動轉大寫比對）
- 免運費折扣：當訂單滿 $599 已自動免運時，此功能主要用於未達門檻的優惠

## 7. 客戶下單流程（前台）

供了解客戶體驗：

```
首頁 (bcs.tw)
  ↓ 點擊商品
商品頁
  → 選擇規格（顏色、尺寸、刻字等）
  → 設定數量（動態顯示價格）
  → 加入購物車
    ↓
購物車 (/cart)
  → 查看商品清單
  → 輸入優惠碼（可選）
  → 選擇運送方式：
      • 到店取件（免費）
      • 指定日期自取（滿 $599 免運，否則 $60）
      • 郵寄到府（滿 $599 免運，否則 $60）
  → 填寫聯絡資訊（姓名/電話/Email/地址）
  → 選擇是否需要「對稿確認」
  → 查看預計出貨日（依數量自動計算）
  → 送出訂單
    ↓
感謝頁 (/thank-you)
  → 顯示訂單編號、應付金額、預計出貨日
  → 顯示匯款資訊（郵局 700，帳號 0031421-0318644，戶名黃詣）
  → LINE 加入好友按鈕（告知訂單編號 + 匯款後五碼）
```

## 8. 常見問題排除 (Q&A)

**Q：訂單成立但我沒收到 LINE 通知？**
A：可能是 GAS Webhook 暫時異常。請直接到後台 `/admin/orders` 查看最新訂單。系統會在 GAS 失敗時記錄到資料庫，不會漏單。

**Q：客人說商品賣完了，但我沒缺貨？**
A：後台商品管理 `/admin/products` 中，確認該商品「啟用」狀態是否為 on。另外確認後台 `/admin/inventory` 中對應原料庫存是否充足（此為原料層控管，非商品本身）。

**Q：客人輸入折扣碼顯示無效？**
A：到 `/admin/coupons` 確認：① 折扣碼是否啟用 ② 是否超過使用次數上限 ③ 是否在有效期內 ④ 客人訂單金額是否達到最低消費門檻。

**Q：我想調整商品在首頁的顯示順序？**
A：到 `/admin/products` → 編輯商品 → 調整「排序權重」數字。數字大的排前面。

**Q：為什麼後台「原料庫存」跟我預期的不同？**
A：後台 `/admin/inventory` 管理的是**原料**（木板、壓克力等），不是商品成品的庫存。如需設定商品成品庫存，需請開發者協助設定 Supabase `inventory` 資料表對應的 SKU 邏輯。

**Q：客人說他選了選項，但訂單顯示英文代碼？**
A：請通知開發者修正對應商品的選項標籤（Label）設定。
